## 2. Noms propres — NE PAS TRADUIRE

### Personnages
Tous les noms de personnages restent en anglais :
- Andrada, Artemisia Sun, Elissa Baird, Cotton, Gargoyle, etc.
- Les titres devant les noms sont traduits ("Haut Hégémon Andrada")

### Lieux et systèmes stellaires
- Askonia, Corvus, Galatia, Hybrasil, Eos, Magec, Mayasura, etc.
- Noms de stations et de planètes : conservés en anglais

### Organisations spécifiques
- Galatia Academy (traduit "Académie de Galatia" si déjà établi, sinon conserver)
- Tri-Tachyon (jamais traduit, jamais "Tri-Tachyon" avec accent)
- Pathers / Luddic Path (la faction est traduite, pas le diminutif "Pather")
- Ministry of Technology Standards (traduire : "Ministère des Normes Technologiques")

### Technologie et noms de vaisseaux
- Noms de classe de vaisseaux : Onslaught, Paragon, Conquest, Dominator, etc. — NON TRADUITS
- Noms d'armes : Hellbore Cannon, Autopulse Laser, etc. — traduits selon le fichier weapon_data.csv
- Noms de systèmes de vaisseaux : Fortress Shield, Burn Drive, etc. — traduits selon ship_systems.csv

## 4. Règles de style

### Vouvoiement
- Le joueur est **toujours vouvoyé** par les PNJ (vous/votre/vos).
- Le joueur peut tutoyer certains PNJ dans des contextes informels (pirates, contacts proches) mais le vouvoiement reste la norme.
- Les officiers s'adressent au joueur par "$PlayerSirOrMadam" traduit en "Monsieur/Madame".

### Ton et registre
- **Ton littéraire soutenu** pour les dialogues narratifs (descriptions de scènes, briefings).
- **Ton professionnel militaire** pour les rapports de combat et les mécaniques de jeu.
- **Ton technique précis** pour les tooltips et descriptions d'équipement.
- Éviter le langage familier sauf dans les dialogues de pirates ou de personnages populaires.

### Conventions typographiques
- **Encodage CSV : UTF-8** (confirmé par test en jeu 2026-03-22). Le vanilla est ASCII pur, le mod utilise UTF-8 pour les accents FR.
- Pas d'accents dans les fichiers strings.json (limitation moteur — vérifié : aucun caractère accentué présent).
- Les fichiers tips.json, descriptions.csv et .faction supportent les accents et caractères spéciaux (dans la limite du jeu de caractères Latin1).
- Les guillemets français `« »` (U+00AB, U+00BB) sont **obligatoires** dans les traductions FR. Bug moteur en cours de résolution (cf. issue #25).
- Pas d'espace insécable avant `:`, `;`, `!`, `?` (contrainte technique du moteur).

### Conjugaison et accords
- Les termes génériques (vaisseau, flotte) suivent le genre grammatical français.
- "Flotte" est féminin : "la flotte ennemie", "elle manœuvre".
- "Vaisseau" est masculin : "le vaisseau ennemi", "il s'échappe".
- Attention aux accords pluriels avec $fleetOrShip (peut être singulier ou pluriel).

### Longueur
- Les traductions FR sont naturellement 15 à 25 % plus longues que l'EN.
- Privilégier la concision dans les tooltips et boutons.
- Les descriptions narratives peuvent être plus longues si nécessaire pour la clarté.

## 5. Variables à préserver

Les variables `$xxx` sont injectées par le moteur du jeu et ne doivent JAMAIS être traduites, déplacées arbitrairement ni supprimées.

### Variables de dialogue courantes

|---|---|---|
| `$playerSirOrMadam` | "Monsieur" ou "Madame" selon le genre du joueur | Dialogues PNJ |
| `$PlayerSirOrMadam` | Idem, majuscule en début de phrase | Dialogues PNJ |
| `$playerName` | Nom du joueur | Dialogues |
| `$playerFleetOrShip` | "flotte" ou "vaisseau" selon la taille | Rapports |
| `$fleetOrShip` | "flotte" ou "vaisseau" de l'ennemi | Rapports |

| `$boardingTaskForceShips` | Liste des vaisseaux de la force d'abordage | Abordage |

## 6. Pièges courants

### Faux amis

| EN | Faux ami FR | Traduction correcte |
|---|---|---|
| actually | actuellement | en réalité / en fait |
| eventually | éventuellement | finalement / à terme |
| to engage | engager (= embaucher) | engager le combat / affronter |
| supplies | provisions | fournitures |
| to salvage | sauver | récupérer |
| casualties | victimes civiles | pertes (militaires) |
| to resume | résumer | reprendre |
| sensitive | sensible (= émotif) | sensible (= délicat) / critique |
| to demand | demander | exiger |
| comprehensive | compréhensif | exhaustif / complet |
| hazard | hasard | danger / risque |
| to support | supporter | soutenir / prendre en charge |
| commodity | commodité | marchandise / denrée |
| fabric | fabrique | tissu (rare en contexte Starsector) |
| to assume | assumer | supposer / présumer |
| patron | patron (= chef) | protecteur / mécène |
| delay | délai | retard |

### Mots techniques à ne pas confondre

| Terme | Attention |
|---|---|
| hull / hold | hull = coque ; hold = soute |
| fleet / float | fleet = flotte ; ne pas confondre |
| crew / crude | crew = équipage ; crude = brut |
| refit / retrofit | refit = rééquipement ; utiliser "réaménagement" ou "équipement" |
| vent / vent (fr) | vent EN = évent (flux) ; ne pas traduire par "vent" |
| deploy / display | deploy = déployer ; display = afficher |
| mount / amount | mount = emplacement (arme) / monter ; amount = quantité |
| ordnance / ordinance | ordnance = armement/munitions ; ordinance = ordonnance |

### Accords particuliers
- "équipe de récupération" : féminin pluriel ("les équipes ont réussi")
- "forces" : féminin pluriel ("vos forces sont")
- "fusilier(s)" : masculin, attention au (s) conditionnel avec les variables numériques
- "membre(s) d'équipage" : masculin, attention au (s) conditionnel

### Cohérence lexicale obligatoire
- Toujours "désengager" (jamais "décrocher" ou "se retirer" pour le terme technique)
- Toujours "aptitude au combat" pour CR (jamais "préparation au combat")
- Toujours "dysfonctionnement" (jamais "panne" ou "bug")
- Toujours "fournitures" pour supplies (jamais "provisions" ou "ressources")
- Toujours "fusiliers" pour marines (jamais "marines" ou "soldats")
- Toujours "carburant" pour fuel (jamais "combustible")
- Toujours "récupération" pour salvage (jamais "sauvetage" sauf contexte personnel)

## Annexe : processus de validation

1. Vérifier la cohérence avec ce glossaire avant toute traduction.
2. Les termes absents du glossaire doivent être proposés et validés avant usage.
3. En cas de doute, privilégier la clarté sur l'élégance.
4. Tester en jeu : certains textes ont des contraintes de largeur d'affichage.
