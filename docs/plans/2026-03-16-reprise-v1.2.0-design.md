# Design : Reprise du projet lang_pack_fr et livrable v1.2.0

**Date** : 2026-03-16
**Auteur** : mipsou + Claude (archi)
**Statut** : Approuve

## Contexte

Le projet Starsector Language Pack FR est compose de deux repos :
- **Prive** (`starsector_lang_pack_fr_private`) : workspace contributeurs (docs, scripts, outils, WIP, guidelines)
- **Public** (`starsector_lang_pack_fr`) : mod distribue (fichiers prets a installer)

Le dernier livrable est la **v1.0.0** (tips.json traduit). Un plan v1.2.0 existait (strings.json + descriptions.csv) mais le projet a ete mis en pause avec du WIP non commite sur les deux repos.

## Etat des lieux

### Repo prive (local feature/translations, 10 commits ahead of dev)
- 6 fichiers de traduction **vides dans le working tree** mais valides en HEAD (ALERTE)
- strings.json : 238/389 lignes (~61% traduit)
- descriptions.csv : 1205 lignes en HEAD (a verifier si traduit ou copie EN)
- tooltips.json, tips.json, ship_names.json, codex.json : traduits en HEAD
- CSV campagne (hullmods, hulls, weapons, shipsystems) : fichiers presents
- 15 missions traduites (dans backups/missions/)
- ~90 fichiers de bruit (backups, .bak, .coverage, logs, etc.)

### Repo public (local feature/translations, 7 commits ahead of dev)
- strings_fr.json : +86 traductions non stagees
- mod_info.json : passage DEV -> stable (non stage)
- tooltips_fr.json, tips_fr.json, ship_names_fr.json : presents
- Bruit : .deb, .wslconfig, localization.old/, jar_extract/

### GitHub
- Tag v1.0.0 sur le public (tips.json)
- Les deux repos en retard vs le local

## Design

### Phase 1 : Securisation immediate
1. Repo prive : `git stash` pour proteger HEAD des modifications destructives
2. Repo public : `git stash` pour sauvegarder le WIP utile
3. Tag `pre-cleanup` sur les deux repos locaux

### Phase 2 : Nettoyage
1. Supprimer le bruit : backups (sauf missions), .coverage, htmlcov, logs, .deb, .wslconfig, jar_extract, *_keys.txt, test_guillemets*, localization.old/
2. Promouvoir missions de `backups/missions/` vers `localization/data/missions/`
3. Ajouter `.gitignore` pour prevenir le retour du bruit

### Phase 3 : Synchronisation
1. Commit propre du nettoyage sur feature/translations (prive)
2. Merge feature/translations -> dev sur les deux repos
3. Push vers GitHub

### Phase 4 : Reprise v1.2.0
1. Finir strings.json (~150 lignes restantes)
2. Verifier/completer descriptions.csv
3. Packaging v1.2.0, release GitHub

## Decisions cles

- **On ne touche PAS au working tree destructif** du prive : HEAD a le bon contenu
- **Le repo prive reste le workspace dev**, le public recoit les fichiers finaux
- **Structure unique** : `localization/data/` est le canonical path pour les traductions
- **Le plan v1.2.0 original est conserve** comme reference mais ce design le supersede

## Hors scope
- Consolidation avec le repo grena (`starsector_french_translation_mod`)
- Traduction du tutoriel (dans starfarer_obf.jar)
- Support de mods tiers
